# Data Structures & Algorithms

A structured repository containing my solutions to various algorithmic problems. This serves as a personal archive of my problem-solving progression, focusing on clean code, algorithmic efficiency, and optimal performance.

## Structure

Problems are automatically organized into their own individual directories. Each directory typically contains:
* The problem statement and constraints.
* The solution code.
* Relevant notes on the approach, time complexity, and space complexity.

## Focus Areas

* **Core Data Structures:** Arrays, Hash Maps, Linked Lists, Trees, Graphs, Heaps, and Tries.
* **Algorithmic Patterns:** Two Pointers, Sliding Window, Fast & Slow Pointers, BFS/DFS, and Binary Search.
* **Advanced Concepts:** Dynamic Programming, Backtracking, Greedy Algorithms, and Bit Manipulation.

## Goals

1. Maintain a consistent problem-solving routine to reinforce algorithmic thinking.
2. Write highly optimized solutions focusing on minimal execution time and memory usage.
3. Build a comprehensive, easily navigable reference library of algorithmic patterns.
